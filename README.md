# 🛠️ Парсер Петрович Pro - Полное руководство

Профессиональный инструмент для автоматического сбора данных о товарах с сайта Петрович с поддержкой 4 уровней категорий, параллельной загрузки и удобным GUI.

---

## 🚀 Быстрый старт

### **Первый запуск (один раз):**

```powershell
# 1. Установить зависимости
pip install customtkinter pillow

# 2. Сгенерировать дерево категорий (займет ~5-10 минут)
python build_full_categories_tree.py

# 3. Запустить GUI
python app_gui.py
```

### **Последующие запуски:**

```powershell
python app_gui.py
```

## 🎯 Использование

### **Шаг 1: Выбор категорий**

1. Откройте `app_gui.py`
2. В древовидной структуре раскройте нужные группы (▶)
3. Выберите категории (☑)
4. При выборе родительской категории автоматически парсятся **все** дочерние

### **Шаг 2: Настройка колонок**

Выберите до 12 колонок для экспорта:
- `Уровень 1-4` - иерархия категорий
- `Артикул` - код товара
- `Название` - наименование
- `Цена` - стоимость (розница)
- `Ед. изм.` - единица измерения
- `Бренд` - производитель
- `Вес` - масса товара
- `Изображение` - ссылка на фото
- `Ссылка` - URL товара

### **Шаг 3: Запуск парсинга**

1. Установите лимит (опционально) - максимум товаров на категорию
2. Нажмите **"ЗАПУСТИТЬ ПАРСИНГ"**
3. Дождитесь завершения
4. Файл автоматически откроется во вкладке "Результаты"

## ⚡ Производительность

### **Генерация дерева категорий:**
- **Категорий:** 157
- **Уровни:** 3 (L1 → L2 → L3)
- **Время:** ~5-10 минут
- **Параллельные потоки:** 10
- **Периодичность:** Один раз (или при обновлении каталога)

### **Парсинг товаров:**
- **Параллельные потоки:** 5 категорий одновременно
- **Ускорение:** В 3-5 раз быстрее последовательного
- **Пример:** 10 категорий = ~5-6 минут (вместо 20)

## 🔧 Настройка

### **Обновление дерева категорий**

Если на сайте появились новые категории:

```powershell
python build_full_categories_tree.py
```

### **Настройка куков**

1. В GUI нажмите **"Настроить Куки"**
2. Вставьте JSON с куками
3. Нажмите **"Сохранить"**

Или вручную отредактируйте файл `Cook`.

### **Параллельная загрузка**

В коде `full_auto_parser_CURL.py` можно изменить параметры:

```python
# Генерация дерева (по умолчанию: 10)
with ThreadPoolExecutor(max_workers=10) as executor:

# Парсинг товаров (по умолчанию: 5)
parallel=True  # False для последовательного режима
```

## 📊 Структура данных CSV

Пример выходного файла:

```csv
level1,level2,level3,level4,article,name,price,unit,brand,weight,image,url
Стройматериалы,Гипсокартон,ГКЛ стандартный,,123456,Лист ГКЛ 2500х1200х12.5,350.50,шт,Knauf,35.5,https://...,https://...
```

### **Поля:**
- **level1-4:** Полная иерархия категорий
- **article:** Артикул Петрович
- **name:** Название товара
- **price:** Цена (розница) в рублях
- **unit:** Единица измерения (шт, м, кг, м2, и т.д.)
- **brand:** Производитель/бренд
- **weight:** Вес в кг
- **image:** URL изображения товара
- **url:** Прямая ссылка на товар

## 📝 Решение проблем

### **Ошибка 403 Forbidden**

Куки устарели. Обновите их:
1. Откройте сайт Петрович в браузере
2. Откройте DevTools (F12) → Network
3. Скопируйте куки из запроса
4. Обновите через GUI или файл `Cook`

Подробнее: `SOLUTION_403_FORBIDDEN.md`

### **GUI не открывается**

Проверьте наличие `categories_full_tree.json`:

```powershell
python build_full_categories_tree.py
```

### **Слишком долго парсится**

- Установите лимит товаров (например, 100)
- Выбирайте конкретные категории вместо всех
- Убедитесь что включен параллельный режим (по умолчанию)

### **Не все товары собираются**

- Убедитесь что выбрана **родительская категория** (она собирает все дочерние автоматически)
- Проверьте лимит товаров (если не установлен = все товары)
- Посмотрите логи в нижней части GUI
